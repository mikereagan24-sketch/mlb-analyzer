FG Daily Sync

One click on a logged-in FanGraphs page updates wOBA projections, wOBA actuals, and RosterResource pitcher roles.

Install

Drag this link to your bookmarks bar:

📊 FG Daily Sync

(If the link doesn't drag, right-click it → "Bookmark Link…".)

How to use

  1. Open fangraphs.com and confirm you're logged in (paid Member).
  2. Click the bookmark.
  3. A small overlay appears in the top-right of the FanGraphs tab and works through three sections: projections (4 requests, fast), actuals (4 requests, fast), roles (30 requests, sequential with a small pause between each — total ~1–2 min).
  4. Sections update independently — a broken roles fetch does not block projections/actuals landing (and vice versa).
  5. When all three finish, hit Close on the overlay.

What it does

  • Projections: fetches the four Steamer RoS split JSONs (bat vs L/R, pit vs L/R) from FanGraphs' internal /api/projections endpoint, converts server-side to the CSV shape the manual upload uses, ingests into the same wOBA table.
  • Actuals: fetches the four two-year split leaderboards (bat vs L/R, pit vs L/R) from FanGraphs' internal /api/leaders/splits/splits-leaders endpoint, ingests the same way.
  • Roles: fetches all 30 RosterResource depth-chart pages sequentially (1.5s between each, human-scale), parses the SP1–SP5 / CL / SU# / MID / IL rows out of the SSR HTML, POSTs a single JSON payload to /api/upload/rr-roles. Server crosswalks each FG player to mlb_id via name + team lookup against team_rosters and upserts pitcher_fg_role with source='rr_sync'.

Caveats

  • The bookmarklet refuses to run on any page that isn't fangraphs.com (it alerts and exits).
  • Requires a logged-in FanGraphs Member session. Without it, projections and actuals will return empty or 403.
  • Runs once daily. Do not schedule or auto-fire — the roles section touches FG 30 times sequentially, which is fine as a human-scale daily press, not as a background loop.
  • The upload target is mlb-analyzer.onrender.com. Edit the APP constant in the source below if pointing at a dev server.
