FG Actuals Bookmarklet

One-click upload of the four split-leaderboard CSVs from FanGraphs to mlb-analyzer.

Install

Drag this link to your bookmarks bar:

📊 Upload FG Actuals

(If the link doesn't drag, right-click it → "Bookmark Link…".)

How to use

  1. Open any fangraphs.com page.
  2. Click the bookmark.
  3. A small overlay appears in the top-right of the FanGraphs tab and walks through the four splits. Each takes 5–10 seconds.
  4. When all four finish, hit Close on the overlay. Your mlb-analyzer actuals are up to date.

What it does

  • Opens four hidden iframes on the FanGraphs tab, one per split (bat vs LHP/RHP, pit vs LHB/RHB).
  • Each iframe renders the splits leaderboard page, which client-side generates an <a download> with a data:-URL CSV.
  • The bookmarklet decodes each CSV and POSTs it as multipart/form-data to mlb-analyzer.onrender.com/api/upload/<data-key>.
  • Failures on one split (timeout, upload error) are shown in the overlay but don't block the other three.
  • Uses groupBy=career and startDate=2024-01-01 so players who split time across seasons aggregate into one row rather than one-per-season.

Caveats

  • The bookmarklet refuses to run on any page that isn't fangraphs.com (it alerts and exits).
  • FanGraphs must render the Export link client-side. If the site ships a CSP that blocks inline JS on that page or wraps the leaderboards in an iframe-busting header, the bookmarklet will time out with no Export link — in which case fall back to the manual refresh button on the Model tab.
  • Logged-in / logged-out: the splits leaderboard CSVs render the same for both, so this works without a FanGraphs Member session. The upload target is always the production host; edit the API constant in the source below to point at a local dev server if needed.
